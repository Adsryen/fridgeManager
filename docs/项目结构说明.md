# 项目结构说明

## 目录结构

```
fridgeManager/
├── app/                    # 应用核心代码
│   ├── __init__.py         # Flask 应用工厂
│   ├── models/             # 数据模型层
│   ├── routes/             # 路由层（控制器）
│   ├── services/           # 业务逻辑层
│   └── utils/              # 工具函数层
├── config/                 # 配置管理
├── docker/                 # Docker 部署
├── docs/                   # 项目文档
├── migrations/             # 数据库迁移
├── scripts/                # 脚本工具
├── static/                 # 静态资源
├── templates/              # HTML 模板
├── tests/                  # 测试代码
└── data/                   # 数据存储（不提交）
```

## 架构设计

### 分层架构

```
┌─────────────────────────────────────┐
│         Presentation Layer          │  templates/ + static/
│         (HTML + JavaScript)         │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│          Routes Layer               │  app/routes/
│         (Controllers)               │  - main.py
│                                     │  - auth.py
│                                     │  - item.py
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│        Business Logic Layer         │  app/services/
│          (Services)                 │  - user_service.py
│                                     │  - item_service.py
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│         Data Access Layer           │  app/utils/database.py
│        (Database Utils)             │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│           Data Layer                │  data/ (SQLite)
│          (Database)                 │
└─────────────────────────────────────┘
```

## 核心模块说明

### 1. app/ - 应用核心

#### app/__init__.py
- Flask 应用工厂模式
- 初始化数据库连接
- 注册蓝图
- 配置错误处理

- **user.py**: 用户数据模型
- **item.py**: 物品数据模型
- 使用 dataclass 定义数据结构

### 2. config/ - 配置管理

### 5. tests/ - 测试代码

- 集成测试
- Pytest 配置

## 设计原则

### 1. 单一职责原则
- 每个模块只负责一个功能
- 路由层只处理 HTTP 请求
- 服务层只处理业务逻辑
- 数据层只处理数据访问

### 2. 依赖倒置原则
- 高层模块不依赖低层模块
- 通过接口/抽象进行交互

### 3. 开闭原则
- 对扩展开放
- 对修改关闭
- 通过蓝图和服务层实现

### 4. 关注点分离
- 表现层、业务层、数据层分离
- 配置与代码分离
- 测试与业务代码分离

## 数据流

### 请求处理流程

```
1. 用户请求
   ↓
2. Routes (路由层)
   - 接收 HTTP 请求
   - 验证参数
   - 调用服务层
   ↓
3. Services (业务逻辑层)
   - 处理业务逻辑
   - 数据验证
   - 调用数据层
   ↓
4. Database Utils (数据访问层)
   - 执行数据库操作
   - 返回结果
   ↓
5. 返回响应
   - JSON 或 HTML
```

## 扩展指南

### 添加新功能

1. **创建数据模型** (app/models/)
   ```python
   @dataclass
   class NewModel:
       field1: str
       field2: int
   ```

2. **创建服务类** (app/services/)
   ```python
   class NewService:
       def __init__(self, db):
           self.db = db
       
       def create(self, data):
           # 业务逻辑
           pass
   ```

3. **创建路由** (app/routes/)
   ```python
   new_bp = Blueprint('new', __name__)
   
   @new_bp.route('/endpoint')
   def handler():
       # 处理请求
       pass
   ```

4. **注册蓝图** (app/__init__.py)
   ```python
   from app.routes import new_bp
   app.register_blueprint(new_bp, url_prefix='/new')
   ```

### 添加新配置

在 `config/settings.py` 中添加配置项：

```python
class Config:
    NEW_CONFIG = os.environ.get('NEW_CONFIG', 'default_value')
```

### 添加数据库表

在 `app/utils/database.py` 的 `_get_connection` 方法中添加：

```python
conn.execute(
    "CREATE TABLE IF NOT EXISTS new_table ("
    "_id TEXT PRIMARY KEY, "
    "field1 TEXT, "
    "field2 INTEGER"
    ")"
)
```

## 最佳实践

1. **代码组织**
   - 相关功能放在同一模块
   - 避免循环导入
   - 使用相对导入

2. **错误处理**
   - 使用 try-except 捕获异常
   - 返回有意义的错误信息
   - 记录错误日志

3. **安全性**
   - 密码加密存储
   - 使用参数化查询
   - 验证用户输入
   - 使用 HTTPS（生产环境）

4. **性能优化**
   - 使用数据库索引
   - 缓存常用数据
   - 异步处理耗时操作

5. **测试**
   - 编写单元测试
   - 编写集成测试
   - 保持高测试覆盖率
