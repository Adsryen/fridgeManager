## 开发环境搭建

### 1. 克隆项目
```bash
git clone https://github.com/Adsryen/fridgeManager.git
cd fridgeManager
```

### 2. 后端环境设置
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 初始化数据库
python scripts/init_db.py
```

### 3. 前端环境设置
```bash
cd frontend

# 安装依赖
npm install

# 或使用 yarn
yarn install
```

## 开发规范

### 代码规范

#### Python 代码规范
- 遵循 PEP 8 规范
- 使用 Black 进行代码格式化
- 使用 isort 进行导入排序
- 使用 flake8 进行代码检查

```bash
# 安装开发工具
pip install black isort flake8

# 格式化代码
black app/
isort app/

# 检查代码
flake8 app/
```

#### TypeScript 代码规范
- 使用 ESLint + Prettier 进行代码检查和格式化
- 遵循 Vue 3 官方风格指南
- 使用 TypeScript 严格模式

```bash
# 检查代码
npm run lint

# 修复代码
npm run lint:fix

# 格式化代码
npm run format
```

### 命名规范

#### 文件命名
- **Python 文件**：使用下划线命名 `user_service.py`
- **Vue 组件**：使用 PascalCase `UserProfile.vue`
- **TypeScript 文件**：使用 camelCase `userService.ts`
- **CSS 文件**：使用短横线 `user-profile.css`

#### 变量命名
- **Python**：使用下划线命名 `user_name`
- **JavaScript/TypeScript**：使用 camelCase `userName`
- **常量**：使用大写下划线 `API_BASE_URL`
- **类名**：使用 PascalCase `UserService`

### Git 提交规范

#### 提交信息格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

#### 类型说明
- **feat**：新功能
- **fix**：修复 bug
- **docs**：文档更新
- **style**：代码格式调整
- **refactor**：代码重构
- **test**：测试相关
- **chore**：构建过程或辅助工具的变动

#### 示例
```bash
feat(auth): 添加用户注册功能

- 实现用户注册接口
- 添加邮箱验证功能
- 更新前端注册表单

Closes #123
```

## 常见问题

### 开发环境问题

**Q: Python 虚拟环境激活失败？**
A: 确保使用正确的激活命令，Windows 使用 `venv\Scripts\activate`，macOS/Linux 使用 `source venv/bin/activate`。

**Q: 前端依赖安装失败？**
A: 尝试清除缓存 `npm cache clean --force`，然后重新安装。

**Q: 数据库连接失败？**
A: 检查数据库文件路径是否正确，确保 `data` 目录存在。

### 代码问题

**Q: TypeScript 类型错误？**
A: 检查类型定义文件，确保导入的类型正确。

**Q: Vue 组件不更新？**
A: 检查响应式数据是否正确使用 `ref` 或 `reactive`。

**Q: API 请求跨域错误？**
A: 确保后端正确配置 CORS，允许前端域名访问。

### 部署问题

**Q: Vercel 部署失败？**
A: 检查构建命令和输出目录配置，确保选择正确的框架预设。

**Q: Docker 容器启动失败？**
A: 检查 Dockerfile 配置和端口映射，查看容器日志排查问题。
